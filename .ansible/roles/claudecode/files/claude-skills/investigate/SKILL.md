# Codebase Investigation Skill

コードベースの調査をTaskエージェントを活用して効率的に行い、結果をまとめます。

## 対応する調査タイプ

### 1. 未使用コード検出
- 特定のモジュール/クラス/メソッドが使われているか
- 未使用のルート、GraphQL operation、API endpoint
- 不要なGem/パッケージ

### 2. 機能の利用箇所調査
- 特定の機能がどこで使われているか
- 影響範囲の特定
- 依存関係のマッピング

### 3. コードパターン調査
- 特定のパターンの使用箇所
- 非推奨APIの利用箇所
- 設定値の使用箇所

## 実行フロー

### 1. 調査計画
ユーザーの質問から調査対象を特定し、必要な検索を洗い出す。

### 2. 並列調査
独立した検索はAgentツール（subagent_type=Explore）で並列実行：
- ファイル検索（Glob）
- コード内検索（Grep）
- 設定ファイルの確認（Read）
- import/require の追跡

### 3. 結果の集約
調査結果を以下のフォーマットでまとめる：

```markdown
## 調査結果: [対象]

### 使用箇所
- ファイル:行番号 - 使用方法

### 依存関係
- どのファイルから参照されているか

### 結論
- 削除可能か / 影響範囲はどこか
```

## 重要ルール

- **変更はしない**: 調査のみ。修正はユーザーの指示を待つ
- **根拠を示す**: 「使われていない」と判断した場合、検索結果を証拠として提示
- **見落としに注意**: 動的呼び出し（send, eval）やメタプログラミングの可能性も考慮
