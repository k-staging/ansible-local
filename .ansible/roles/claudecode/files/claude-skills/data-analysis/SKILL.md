# Data Analysis Skill

データ分析用のSQLクエリを、スキーマ検証→サンプル実行→本番クエリの段階的フローで作成します。

## 実行フロー（必ずこの順序で）

### Step 1: スキーマ確認（推測しない）
対象テーブルの実際のスキーマを確認する：
- `db/schema.rb` または `db/structure.sql` を読む
- カラム名、型、enum定義を**実データから取得**
- **絶対にカラム名を推測しない**

```bash
# Railsプロジェクトの場合
grep -A 30 'create_table "対象テーブル"' db/schema.rb
```

### Step 2: クエリ作成
- ユーザーの分析目的に合ったクエリを作成
- カラム名はStep 1で確認したものだけを使用
- 集計の単位（日次/月次、ユニークユーザー等）を明確にする

### Step 3: サンプル検証（可能な場合）
- 1日分など小さなサンプルでクエリを実行
- 結果の妥当性をチェック：
  - 数値が異常に大きい/小さいなら原因を調査
  - COUNT vs COUNT(DISTINCT ...) の違いに注意
  - NULLの扱いを確認

### Step 4: 本番クエリ実行
- サンプル検証で問題なければフル期間で実行
- 結果をMarkdownテーブルで整理

### Step 5: 結果の提示
```markdown
## 分析結果: [テーマ]

### クエリ
（実行したSQL）

### 結果
| 月 | 指標A | 指標B |
|---|---|---|

### 考察
- トレンドや特徴を簡潔に
```

## 重要ルール

- **カラム名は必ずスキーマから取得**: 推測は最大のフリクション原因
- **集計ロジックを明示**: 「DAU = 日ごとのユニークユーザー数の月平均」等
- **数値の妥当性を自己チェック**: ユーザーに見せる前に「この数値は妥当か？」を考える
- BigQueryの場合は `INFORMATION_SCHEMA` やテーブルプレビューでスキーマ確認
